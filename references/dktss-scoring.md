# DKTSS 评分系统详解

## 概述

DKTSS（Detection-Kill-Triage-Score-System）是一个针对代码审计场景的漏洞优先级评分系统。相比 CVSS，DKTSS 考虑了实际利用的摩擦度、武器化程度和版本因素，更贴近安全团队的修复决策。

## 公式

```
Score = min(10, Base - Friction + Weapon + Ver)
```

## Base 分值表

| 漏洞类型 | Base | 说明 | 子类差异 |
|----------|------|------|----------|
| RCE | 10 | 直接代码执行 | 反序列化/SSTI/表达式注入/命令执行 |
| SQL 注入（写权限） | 8 | 可修改/删除数据 | UPDATE/DELETE/INSERT |
| SQL 注入（读权限） | 6 | 仅可读取数据 | SELECT |
| SSRF（内网可达） | 7 | 可访问内网服务 | 有内网服务发现 |
| SSRF（仅外网） | 4 | 仅可发起外部请求 | 无内网利用面 |
| 文件上传（可执行） | 9 | 上传并执行代码 | Web 根目录 |
| 文件上传（不可执行） | 5 | 仅可存储文件 | 非 Web 目录 |
| 文件读取（敏感文件） | 7 | 可读配置文件/密钥 | /etc/passwd, .env |
| 文件读取（普通文件） | 5 | 可读非敏感文件 | 公开资源 |
| XXE（有回显） | 8 | 可直接读取文件 | 反射型 XXE |
| XXE（无回显） | 6 | 需外带数据 | 盲 XXE |
| 认证绕过 | 8 | 绕过认证获得权限 | 水平/垂直越权 |
| CSRF（敏感操作） | 6 | 可执行敏感操作 | 转账、删数据 |
| CSRF（非敏感操作） | 4 | 仅可执行低敏感操作 | 改昵称 |
| XSS（存储型） | 7 | 持久化恶意脚本 | 评论、资料 |
| XSS（反射型） | 5 | 需诱导点击 | 搜索、重定向 |
| 弱加密 | 5 | 可被破解的加密 | MD5/DES/ECB |
| 硬编码密钥 | 4 | 密钥暴露在代码中 | API Key/密码 |
| 开放重定向 | 3 | 可跳转到恶意站点 | phishing |
| 信息泄露 | 4 | 暴露敏感信息 | 堆栈追踪/配置 |
| CRLF 注入 | 4 | 可注入 HTTP 头 | 头注入/响应分割 |
| LDAP 注入 | 7 | 可绕过 LDAP 认证 | 目录服务 |

## Friction 扣分表

### 访问路径

| 条件 | 扣分 |
|------|------|
| 公开可达（无需认证） | 0 |
| 需要普通用户认证 | -1 |
| 需要管理员/特权认证 | -3 |
| 需要内网访问 | -2 |

### 权限阈值

| 条件 | 扣分 |
|------|------|
| 无需权限 | 0 |
| 任意注册用户 | -1 |
| 特定角色/组 | -2 |
| 仅管理员/超级用户 | -3 |

### 交互复杂度

| 条件 | 扣分 |
|------|------|
| 自动触发（无需交互） | 0 |
| 需要用户点击链接 | -1 |
| 需要多步时序攻击 | -3 |

### Friction 计算

```
Friction = max(访问路径扣分, 权限阈值扣分) + 交互复杂度扣分
```

访问路径和权限阈值取较大值（因为两者通常是重叠的），交互复杂度独立累加。

## Weapon 加分表

| 级别 | 加分 | 说明 |
|------|------|------|
| 成熟 EXP 工具公开可用 | +1 | 如 Log4Shell EXP、Fastjson EXP |
| 有 PoC 但未武器化 | 0 | 有概念验证代码 |
| 理论可行，无公开 PoC | -2 | 需要复杂的攻击链 |

## Ver 版本因子

| 情况 | 加分 | 说明 |
|------|------|------|
| 当前版本，无补丁 | 0 | 最新版仍受影响 |
| 新版本已修复，目标仍为旧版 | +1 | 存在可用补丁但未应用 |
| 仅影响已弃用版本 | -1 | 升级即可修复 |

## 示例计算

### 示例 1: 公开 SQL 注入

```
Base = 6（读权限 SQL 注入）
Friction = 0（公开可达）+ 0（无需交互）= 0
Weapon = 1（SQLMap 成熟工具）
Ver = 0（当前版本）
Score = min(10, 6 - 0 + 1 + 0) = 7 (High)
```

### 示例 2: 后台管理 SQL 注入

```
Base = 6（读权限 SQL 注入）
Friction = 3（管理员认证）+ 0（无需交互）= 3
Weapon = 1（SQLMap 可用）
Ver = 0
Score = min(10, 6 - 3 + 1 + 0) = 4 (Medium)
```

### 示例 3: 反序列化 RCE

```
Base = 10（RCE）
Friction = 0（公开可达）
Weapon = 1（成熟 EXP）
Ver = 0
Score = min(10, 10 - 0 + 1 + 0) = 10 (Critical)
```

## DKTSS 与 CVSS 的关系

| 场景 | DKTSS | CVSS 3.1 | 差异说明 |
|------|-------|----------|----------|
| 后台 SQLi | 4 (Medium) | 8.8 (High) | DKTSS 考虑认证摩擦，更合理 |
| 反序列化 RCE | 10 (Critical) | 9.8 (Critical) | 一致 |
| 反射型 XSS | 5 (Medium) | 6.1 (Medium) | 接近 |
| 存储型 XSS | 7 (High) | 6.1 (Medium) | DKTSS 考虑持久化影响 |
